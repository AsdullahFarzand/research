## A

### Adversarial Machine Learning
The study of attacks on machine learning systems and defenses against them. Includes evasion attacks, poisoning attacks, and model extraction.

### Alert Fatigue
A condition where security analysts become desensitized to security alerts due to high volume, leading to missed genuine threats.

### Anomaly Detection
A technique for identifying patterns that deviate from expected behavior, often used to detect novel or unknown threats.

### APT (Advanced Persistent Threat)
A sophisticated, long-term cyberattack typically conducted by nation-states or organized crime groups targeting specific organizations.

### Asset Inventory
A comprehensive catalog of all hardware, software, and data assets within an organization's IT environment.

### Attack Surface
The sum of all points where an attacker could attempt to enter or extract data from an environment.

### Attack Vector
The method or pathway used by an attacker to gain unauthorized access to a system or network.

---

## B

### Baseline
A reference point of normal system or network behavior used to identify deviations that may indicate security incidents.

### Behavioral Analysis
Security technique that monitors and analyzes user or system behavior to detect anomalies or malicious activity.

### Blue Team
The defensive security team responsible for maintaining security posture and responding to attacks.

### Bug Bounty
A program offering rewards to security researchers who discover and responsibly disclose vulnerabilities.

---

## C

### C2 (Command and Control)
Infrastructure used by attackers to communicate with and control compromised systems.

### CIA Triad
The three fundamental principles of information security: Confidentiality, Integrity, and Availability.

### Compensating Control
An alternative security measure implemented when a primary control cannot be applied.

### Concept Drift
The phenomenon where the statistical properties of data change over time, affecting machine learning model performance.

### CVE (Common Vulnerabilities and Exposures)
A standardized identifier for publicly known security vulnerabilities.

### CVSS (Common Vulnerability Scoring System)
A standardized framework for rating the severity of security vulnerabilities on a scale of 0-10.

### CWE (Common Weakness Enumeration)
A categorization system for software security weaknesses and vulnerabilities.

---

## D

### DAST (Dynamic Application Security Testing)
Security testing that analyzes applications while they are running to find vulnerabilities.

### Data Exfiltration
The unauthorized transfer of data from an organization's network to an external location.

### Deep Learning
A subset of machine learning using neural networks with multiple layers for complex pattern recognition.

### Defense in Depth
A security strategy employing multiple layers of protection to defend against various attack vectors.

### Detection Rate
The percentage of actual threats correctly identified by a security system.

### Digital Forensics
The process of collecting, preserving, and analyzing digital evidence for legal or investigative purposes.

---

## E

### Endpoint
Any device connected to a network, such as computers, mobile devices, or IoT devices.

### Endpoint Detection and Response (EDR)
Security solutions that monitor endpoints for suspicious activity and enable response to threats.

### Enumeration
The process of gathering information about a target system, including users, shares, and services.

### Evasion Attack
A technique used to bypass security controls or detection systems.

### Exploit
Code or technique that takes advantage of a vulnerability to compromise a system.

### Exploitation
The act of using a vulnerability to gain unauthorized access or cause unintended behavior.

---

## F

### False Negative
A security event that should have been detected but was missed (Type II error).

### False Positive
A security alert triggered by benign activity mistakenly identified as malicious (Type I error).

### Feature Engineering
The process of selecting and transforming variables for use in machine learning models.

### Firewall
A network security device that monitors and controls incoming and outgoing network traffic.

### Forensic Investigation
The systematic process of collecting and analyzing evidence following a security incident.

### Fuzzing
A testing technique that provides random or malformed input to discover software vulnerabilities.

---

## G

### Gap Analysis
The process of identifying differences between current security posture and desired state.

### Governance
The framework of policies, procedures, and controls for managing organizational security.

---

## H

### Heuristic Analysis
Detection method using rules and patterns to identify potentially malicious behavior.

### Honeypot
A decoy system designed to attract and detect attackers while gathering intelligence on their methods.

### Human-AI Collaboration
The integration of human expertise and artificial intelligence capabilities for improved security operations.

---

## I

### IDS (Intrusion Detection System)
A system that monitors network traffic for suspicious activity and alerts security personnel.

### Incident Response (IR)
The organized approach to addressing and managing security incidents.

### Indicator of Compromise (IOC)
Evidence that a security breach has occurred, such as unusual network traffic or file changes.

### IPS (Intrusion Prevention System)
A system that actively blocks detected threats in addition to monitoring.

---

## J

### Just-In-Time Access
Security model providing temporary, time-limited access to resources only when needed.

---

## K

### Kill Chain
A model describing the stages of a cyberattack from reconnaissance to objective completion.

### Key Performance Indicator (KPI)
Measurable values used to evaluate the effectiveness of security operations.

---

## L

### Lateral Movement
The technique attackers use to move through a network after gaining initial access.

### Log Analysis
The process of examining log data to identify security events and anomalies.

### Living off the Land (LotL)
Attack technique using legitimate tools already present in the target environment.

---

## M

### Machine Learning (ML)
A subset of AI enabling systems to learn patterns from data without explicit programming.

### Malware
Software designed to cause damage, gain unauthorized access, or perform other malicious activities.

### Mean Time to Detect (MTTD)
The average time required to identify a security incident.

### Mean Time to Respond (MTTR)
The average time required to respond to and contain a security incident.

### MITRE ATT&CK
A knowledge base of adversary tactics and techniques based on real-world observations.

---

## N

### Network Segmentation
The practice of dividing a network into smaller segments to limit attack spread.

### NIST (National Institute of Standards and Technology)
U.S. government agency that develops cybersecurity standards and guidelines.

---

## O

### OSINT (Open Source Intelligence)
Intelligence gathered from publicly available sources.

### OWASP (Open Web Application Security Project)
A nonprofit organization focused on improving software security.

---

## P

### Patch Management
The process of acquiring, testing, and installing software updates to fix vulnerabilities.

### Penetration Testing (Pen Testing)
Authorized simulated attacks on systems to evaluate security posture.

### Phishing
Social engineering attack using deceptive communications to steal credentials or install malware.

### Playbook
Documented procedures for responding to specific types of security incidents.

### Post-Exploitation
Activities performed after successfully compromising a system, such as privilege escalation.

### Privilege Escalation
The act of gaining higher-level permissions than originally granted.

---

## Q

### Quarantine
The isolation of suspicious files or systems to prevent potential damage.

---

## R

### Ransomware
Malware that encrypts files and demands payment for decryption keys.

### Reconnaissance
The initial phase of an attack involving information gathering about the target.

### Red Team
Offensive security team that simulates real-world attacks to test defenses.

### Remediation
The process of fixing identified vulnerabilities or security issues.

### Risk Assessment
The process of identifying, analyzing, and evaluating security risks.

---

## S

### Sandbox
An isolated environment for safely executing and analyzing suspicious code.

### SAST (Static Application Security Testing)
Security testing that analyzes source code without executing the application.

### SCA (Software Composition Analysis)
Analysis of third-party and open-source components for known vulnerabilities.

### SIEM (Security Information and Event Management)
Platform that aggregates and analyzes security data from multiple sources.

### SOC (Security Operations Center)
A centralized facility for monitoring, detecting, and responding to security incidents.

### SOAR (Security Orchestration, Automation and Response)
Technologies enabling automated security operations and incident response.

### Social Engineering
Manipulation techniques used to deceive people into divulging sensitive information.

### Spear Phishing
Targeted phishing attacks directed at specific individuals or organizations.

### Supply Chain Attack
Attack that targets less-secure elements in an organization's supply chain.

---

## T

### Threat Hunting
Proactive searching for threats that may have evaded existing security controls.

### Threat Intelligence (TI)
Information about current and emerging threats used to inform security decisions.

### Threat Modeling
The process of identifying potential threats and vulnerabilities in systems.

### Triage
The process of prioritizing security alerts based on severity and impact.

---

## U

### UEBA (User and Entity Behavior Analytics)
Security technology that uses analytics to detect anomalous user or entity behavior.

---

## V

### Vulnerability
A weakness that can be exploited to compromise system security.

### Vulnerability Assessment
The process of identifying and classifying vulnerabilities in systems.

### Vulnerability Management
The ongoing process of identifying, evaluating, and addressing vulnerabilities.

### Vishing
Voice phishing - social engineering attacks conducted via telephone.

---

## W

### Web Application Firewall (WAF)
Security solution that filters and monitors HTTP traffic to web applications.

### Whaling
Phishing attacks specifically targeting high-profile executives.

---

## X

### XDR (Extended Detection and Response)
Security solution that integrates multiple security products for comprehensive threat detection.

### XSS (Cross-Site Scripting)
A web vulnerability allowing attackers to inject malicious scripts into web pages.

---

## Y

*No terms currently defined for this letter.*

## Z

### Zero-Day
A vulnerability unknown to the vendor with no available patch.

### Zero Trust
Security model requiring verification from everyone attempting to access resources.
